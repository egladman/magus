Query syntax and views

Each view answers a developer question. The search box speaks the same grammar as magus query. Click an example to run it.

QuestionViewCLI equivalent
What breaks if I change this?magus explain <node-id-or-name>
Why does A depend on B?magus path <a> <b>
What's slow?magus graph deps -o json
What's a hub?magus query kind:target
What's dead?magus query -kind:project

The filter box speaks the same grammar as magus query. Terms are combined with AND and matched case-insensitively.

Free text: match a node's id, label, or doc.
Only nodes of one kind: project, target, spell, op, charm, module, method, import, function, file, doc, rationale, diagnostic, symbol. (Or click a legend color.)
All targets in the graph. In a target graph (magus describe graph -o json), anchor targets - those nothing depends on within their project - are shown with a ring outline.
A project node and its targets.
Nodes touching a uses edge (any of depends_on, contains, uses, calls, imports, references, documents, rationale_for).
Substring match on the node id (e.g. target:.:build).
A leading - negates a term - exclude these.
Quote a value that contains spaces to keep it one term.
Combine terms; all must match. This lists targets whose id doesn't contain generate.
SCIP code-symbol nodes (from magus refs), when present in the loaded graph.

A query dims non-matching nodes and hides edges except those between two matches, so the matching subgraph stands out. The same string works as magus query "<terms>" on the command line, or as a #q= deep link on this page.

Local graph: double-click any node (on the canvas or a pill) to isolate its neighborhood; [/] change depth, Esc clears. Fit frames everything (or the current matches).

      Loading a graph by URL

      magus graph open builds these links for you; the fragment (after #) is never sent to the server, so an inline graph stays local. The schemes are:

      #data=<blob>
      An inline graph: the node-link JSON, gzipped then base64url-encoded. Decoded in the browser. Bounded by the browser's URL length.
      #src=<url>
      A URL-encoded address the page fetches the JSON from - a loopback server (magus graph open --serve) or any CORS-enabled address, e.g. a committed graph.json's raw link. The same #src= the playground uses.
      #live=127.0.0.1:7391&token=<bearer>
      Connect to a running daemon for a live workspace view. The host must be literally 127.0.0.1 or [::1] (not localhost). Token consumed on load, stripped from the URL, stored in session storage. Use magus graph open --live to build this link. Not available in Safari (see browser bridge docs).
      #node=<id>
      Select and center a node on load, e.g. #node=spell:go.
      #q=<query>
      Apply a query on load (same grammar as magus query), e.g. #q=kind:spell.
      #view=<id>&node=<id>[&to=<id>]
      Restore a view on load: #view=blast&node=spell:go highlights blast dependents; #view=trace&node=a&to=b draws the path.
      #preset=<id>
      Apply a color preset on load: spell, project, or depth.

      Combine them with &: #data=...&node=target:.:build. A raw graph.json (from magus graph export -o json) also loads via the Open file button or drag-and-drop.
