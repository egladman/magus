magus

A fast cross-platform task orchestrator for polyglot (mono)repos.

Magus computes affected projects from your changes, caches the results, and runs the minimum rebuild set after a change.

Single binary. Code as configuration. Statically typed. No second toolchain.

Documentation

Full docs live at https://eli.gladman.cc/magus/ (source). Start there for targets, spells, charms, CI, remote caching, the daemon, MCP, telemetry, and the rest.

Install

magus ships as a single self-contained binary. See the Download guide.

Development

Building magus itself. Requires Go 1.25+; mise is recommended for the pinned toolchain (see mise.toml).

git clone https://github.com/egladman/magus
cd magus
go build -o magus ./cmd/magus

Run the tests:

go test ./...

Or, once magus is built, via its own targets:

magus run test
magus run ci

Project layout

Path Purpose
cmd/magus/ CLI entry point
cmd/magus-utils/ release signing helper
internal/ core engine (affected set, cache, sandbox, daemon, MCP)
gopherbuzz/ the embedded Buzz interpreter
std/ magusfile stdlib (fs, os, http, ...)
spells/ built-in language spells (go, rust, typescript, ...)
docs/ documentation source (rendered by magus run generate website)
website/ docs-site generator (Buzz-based static site)
