Tour

A guided walk through magusfiles and Buzz in five steps. Open any example in the Playground to run and edit it.

  1. 1

    Hello, magus

    A magusfile is a build definition; each exported function is a runnable target.

    import "magus";

export fun greet(args: [str]) > void {
    magus.info("hello from magus");
}

  4. 4

    Buzz is a real language

    Variables, functions, recursion, closures - evaluated live in your browser.

    import "magus";

// Recursion, the classic. `buzz fibo(30)` computes 832040 in a real bytecode VM.
fun fibo(n: int) > int {
    if (n < 2) { return n; }
    return fibo(n - 1) + fibo(n - 2);
}

  5. 5

    A real magusfile

    The full multi-toolchain pipeline: generate, format, lint, build, test, release.

    import "magus";                 // project registration, targets, dependency edges
import "magus/spell/buf";       // protobuf toolchain: buf-generate, buf-lint, buf-format
import "magus/spell/go";        // Go toolchain: go-build, go-test, go-vet, go-fmt, ...
import "magus/spell/ts";        // TypeScript toolchain: tsc, eslint, prettier, vitest
import "magus/spell/docker";    // container image: docker-build, hadolint

final VERSION: str = "1.4.0";
