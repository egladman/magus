Graph Explorer

See how everything in a magus workspace connects: projects, targets, spells, and the rest.

Runs in your browser; nothing leaves your machine. magus graph open loads your own workspace here. Install magus.

Query syntax

The filter box speaks the same grammar as magus query. Terms are combined with AND and matched case-insensitively; click an example to run it.

Free text: match a node's id, label, or doc.
Only nodes of one kind: project, target, spell, op, charm, module, method, import, function, file, doc, rationale, diagnostic. (Or click a legend color.)
A project node and its targets.
Nodes touching a uses edge (any of depends_on, contains, uses, calls, imports, references, documents, rationale_for).
Substring match on the node id (e.g. target:.:build).
A leading - negates a term - exclude these.
Quote a value that contains spaces to keep it one term.
Combine terms; all must match. This lists targets whose id doesn't contain generate.

A query dims non-matching nodes and hides edges except those between two matches, so the matching subgraph stands out. The same string works as magus query "<terms>" on the command line, or as a #q= deep link on this page.

Lenses (mirroring magus graph stats): the highest-degree "god" nodes, nodes with no edges. Local graph: double-click any node (on the canvas or a pill) to isolate its neighborhood; [/] change depth, Esc clears. Fit frames everything (or the current matches).

      Loading a graph by URL

      magus graph open builds these links for you; the fragment (after #) is never sent to the server, so an inline graph stays local. The schemes are:

      #data=<blob>
      An inline graph: the node-link JSON, gzipped then base64url-encoded. Decoded in the browser. Bounded by the browser's URL length.
      #src=<url>
      A URL-encoded address the page fetches the JSON from - a loopback server (magus graph open --serve) or any CORS-enabled address, e.g. a committed graph.json's raw link. The same #src= the playground uses.
      #node=<id>
      Select and center a node on load, e.g. #node=spell:go.
      #q=<query>
      Apply a query on load (same grammar as magus query), e.g. #q=kind:spell.

      Combine them with &: #data=...&node=target:.:build. A raw graph.json (from magus graph export -o json) also loads via the Open file button or drag-and-drop.
