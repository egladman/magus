# BZZ1001: undefined name

The program referenced a name - a variable or function - that is not defined in
the current scope.

```text
[BZZ1001] buzz: line 6:1: undefined: greet
  see: .../BZZ1001.md
```

## Why

Every name must be declared before use (functions may be referenced before their
declaration within a file; locals may not). A name that resolves to nothing is
usually a typo, a forgotten declaration, or a name that lives in another module.

## Resolution

- Check the spelling against the declaration.
- If it lives in another module, import that module and qualify the call.
- If it IS in the module you imported but still reads as undefined, the module
  may not `export` it - exports-only visibility hides an unexported name. Add
  `export` to the declaration in that module.
