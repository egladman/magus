# BZZ1005: type mismatch

A value's type does not match the type required in that position - an assignment
to a typed variable, a function's declared return, a `*>` yield, or an operand of
an operator.

```text
[BZZ1005] buzz: line 8:9: cannot assign str to int variable "count"
  see: .../BZZ1005.md
```

## Resolution

Make the types line up: convert the value (`int(x)`, `str(x)`), fix the
variable/return annotation to the type you actually produce, or correct the
expression so its operands share a type the operator accepts.
