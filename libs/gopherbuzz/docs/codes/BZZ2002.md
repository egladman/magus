# BZZ2002: fiber misuse

`resume` or `resolve` was used incorrectly. Both operate on a fiber value and
have rules the call broke:

- called with no argument,
- called with a value that is not a fiber,
- called on a fiber that is already running (a recursive resume/resolve).

```text
[BZZ2002] buzz: cannot resume a running fiber (recursive resume)
  see: .../BZZ2002.md
```

## Resolution

Pass a fiber value that is suspended (not the one currently executing). A fiber
cannot resume or resolve itself while it is running; restructure so the
resume/resolve happens from outside that fiber.
