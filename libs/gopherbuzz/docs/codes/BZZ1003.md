# BZZ1003: condition must be bool

The condition of an `if`, `while`, `for`, or `do-until` evaluated to something
other than `bool`. Unlike some languages, buzz does not coerce numbers, strings,
or null to a truthy value - a condition must be a genuine boolean.

```text
[BZZ1003] buzz: line 5:4: if condition must be bool, got int
  see: .../BZZ1003.md
```

## Resolution

Make the condition a boolean expression: compare explicitly (`n != 0`,
`s != ""`, `x == null`) rather than relying on truthiness.
