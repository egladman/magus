Filter syntax

The filter box narrows the log to matching lines. Terms combine with AND and match case-insensitively; the whole filter is shareable as a #q= link fragment.

compile
Free text: match any command or output line containing it.
step:<text>
Match a step (command/output) line containing the text - the same as free text, written explicitly.
target:<text>
Keep lines under a target whose label contains the text, e.g. target:build.
status:pass|fail|cached|running
Keep only steps with that result.

Combine them: status:fail target:build compile shows failing build steps that mention "compile".

Loading a log by URL

magus query output <ref> --open builds these links for you. Everything rides the fragment (after #), which the browser never sends to a server, so nothing about the run - not even its ref - ever leaves your machine.

#ref=<id>
The target-output reference id shown for identity, e.g. #ref=ref1a2b3c.
#data=<blob>
The captured output: the raw bytes, gzipped then base64url-encoded, decoded in the browser. Bounded by the browser's URL length; for a very large log, print it instead with magus query output <ref>.
#src=<url>
A URL-encoded address the page fetches the log text from (a loopback server or any CORS-enabled address).
#live=<host:port>&token=<t>
A loopback SSE endpoint printed by magus run --live; the run streams in as it happens. The bearer token is stripped from the URL once the page reads it.
